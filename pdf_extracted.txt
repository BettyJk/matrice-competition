Step-by-Step Description of the Project Process
Jouak Bouthayna
Introduction
This report explains, in very simple and detailed steps, what I am going to do during my project. The goal
of the project is to create an intelligent system that helps engineers do benchmarking and implantation work
automatically. Today, engineers do many tasks manually in CATIA. My project will automate these tasks
using Artificial Intelligence.
I describe below every step of the process, from the beginning until the end, in the simplest possible way.
Step 1: Understanding the Work of Engineers
•First, I learn what mechanical engineers do in benchmarking.
•I look at their screenshots, their reports, and how they work.
•I understand how they check distances, collisions, packaging limits, cable routing, and design con-
straints.
•I also learn what the implantation team does: they check if a component fits inside the car environment.
Step 2: Collecting the Input Files
•Engineers give STL files. An STL file is a 3D mesh of a part.
•These STL files are the only input the AI system needs.
•I collect these STL files and store them in a clean folder.
•I also collect packaging rules (like minimum distance rules).
Step 3: Preparing the STL Files
•I load each STL file in Python.
•I clean the mesh (remove noise, fix shape problems).
•I convert the mesh into a point cloud, because AI models understand point clouds better.
•I normalize the shape so all parts have a similar scale.

Step 4: Training the Machine Learning Model
•I train a 3D machine learning model, like PointNet or MeshCNN.
•This model learns to understand the 3D geometry of the part.
•It learns to read the shape and produce measurements like:
–distances,
–clearances,
–collisions,
–areas of risk (critical zones).
•This replaces the manual measurements engineers do in CATIA.
Step 5: Extracting All Measurements Automatically
•I run the trained model on each new STL file.
•The model outputs all measurements automatically.
•These measurements will be used later for analysis.
Step 6: Applying Packaging Rules
•I load the rules used by engineers, such as:
–minimum distance between parts,
–minimum clearance for cables,
–alignment constraints,
–allowed space zones.
•I apply these rules to the measurements extracted by the model.
•This allows the system to detectpoints dursautomatically.
Step 7: Detecting Points Durs
•The system checks every measurement.
•If a rule is broken, the system marks apoint dur.
•Examples of points durs:
–two parts are too close,
–a cable routing is impossible,
–a connector is not aligned,
–a component collides with another.
2

Step 8: Generating Implantation Scenarios
•When the design has problems, the system generates scenarios.
•A scenario is a new possible way to place the component.
•The AI tries simple modifications like:
–moving the part a few millimeters,
–rotating the part,
–changing cable routing,
–testing alternative orientations.
•For each scenario, the AI checks if it respects the rules.
Step 9: Evaluating All Scenarios
•Each scenario is tested with packaging rules.
•The AI marks each one as:
–OK (scenario is good),
–NOK (scenario still has problems).
•The AI also gives a score for each scenario.
Step 10: Using the RAG Agent for Reasoning
•The RAG agent (Retrieval-Augmented Generation) is used here.
•It reads all the measurements and scenario results.
•It reads the packaging rules again.
•It explains clearly:
–why the design is OK or NOK,
–why a scenario works or does not work,
–what the best scenario is,
–what the engineering justification is.
Step 11: Proposing Solutions Automatically
•When the design is NOK, the AI proposes solutions.
•Solutions can be:
–move the part,
3
--- Page 4 ---
–rotate the part,
–change cable direction,
–increase a clearance,
–modify the connector angle.
•These solutions are very similar to what engineers do manually.
Step 12: Generating the Final Report Automatically
•The system gathers everything:
–measurements,
–points durs,
–scenarios,
–OK/NOK results,
–proposed solutions,
–explanations from the RAG agent.
•It creates a final report.
•The report looks like the ones engineers create manually.
•The report is clear, structured and ready to use.
Conclusion
This project builds a complete intelligent system that can read 3D parts, analyze them, detect problems,
generate solutions, create scenarios and produce a full report. It replaces many manual tasks that mechanical
engineers do today.
